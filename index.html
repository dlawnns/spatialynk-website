SpatiaLynk — Multi-Level & Explainable Location Recommendations

SpatiaLynk is a spatial-textual recommender system that helps users discover meaningful places by combining map location, user preferences, and place descriptions, with clear explanations for every recommendation.

This website introduces our Final Year Project, documents our progress, and serves as a marketing page for our system prototype.

About Us

Problems with modern platforms and our objectives

Problems We See

  • Fragmented signals: Platforms treat spatial vs textual signals separately.
  • Low explainability: Users don’t know why places are recommended.
  • One-size-fits-all: Results ignore map level and user intent.

Project Objectives

High-level goals of FYP-25-S4-08

  • Design and implement a multi-level spatial-textual recommendation system that can operate at region, district, and POI levels.
  • Integrate explainable AI so that users can see why a place was recommended based on their preferences and past behaviour.
  • Support real-time recommendation using streaming data from user interactions and spatial queries.
  • Provide a map-based web and mobile interface that is intuitive for different user roles (end users, system admin, business partners).
  • Evaluate the system’s functional, non-functional, and ML performance, including cold-start and explanation fidelity.

System Overview

Architecture and main components in simple terms

High-Level Architecture

SpatiaLynk is organised into several main layers to support real-time, explainable location recommendations:

  • Client Layer: Web and mobile applications where users search, view maps, and receive recommendations.
  • Data Ingestion Layer: Collects real-time user events (searches, likes, visits) and static data (POIs, users).
  • Model Training Layer: Trains ML models such as XGBoost and Graph Neural Networks on spatial-textual data.
  • Real-Time Recommendation Layer: Generates, ranks, and explains recommendations for each user request.

The system uses a combination of PostgreSQL/PostGIS, Neo4j, and a data lake (AWS S3 + Snowflake) to store and process data.

User Roles Supported

The system is designed for multiple user roles, each with its own set of use cases:

  • Unregistered User: Browse public content and view POIs without logging in.
  • Registered Free-Tier User: Create an account, set preferences, receive recommendations, give feedback.
  • Registered Premium User: Access advanced features like itinerary generation and saved POI folders.
  • Business Partner: Upload and manage business listings as Points of Interest (POIs).
  • System Admin: Manage system objects, monitor system health, and moderate content.

System Marketing & Value Proposition

Unique selling points, features, and user benefits

Unique Selling Propositions (USP)

What makes SpatiaLynk stand out compared to typical recommendation systems

  • Multi-Level Spatial Recommendations: Adapts results across different map levels (region → district → specific POIs).
  • Explainable AI: Every recommendation is accompanied by a clear, user-friendly explanation template (e.g. “Recommended because you liked quiet cafes nearby and often search in this district”).
  • Hybrid Spatial + Textual Intelligence: Combines proximity, categories, descriptions, and user reviews instead of relying on only one data type.
  • Graph-Based Reasoning: Uses a graph database to trace meaningful paths between users and POIs for better explanations.
  • Research-Grade but Practical: Designed as an academic prototype that can be extended into real-world smart city and tourism use cases.
Key Feature
Multi-Level Recommendations

The system can recommend at different spatial levels. When users zoom out, they see recommended districts or regions. When they zoom in, the system refines recommendations to specific POIs.

Key Feature
Explainable Recommendations

For each recommended POI, SpatiaLynk generates a short explanation based on graph paths and user preferences. This increases trust and helps users understand why the system suggested a place.

Key Feature
Personalised Ranking

Candidate POIs are first generated using embeddings and graph models, then ranked by an XGBoost model using features such as distance, popularity, user interests, and past interactions.

Premium Feature
Itinerary & Saved Collections

Premium users can generate day itineraries and save recommended POIs into custom folders (e.g. “Weekend Hangouts”, “Study Spots”), improving planning efficiency.

User Benefits

  • Time Savings: Reduces time spent searching for suitable places by combining filters, preferences, and location into a single system (estimated 30–50% faster than manual search).
  • Higher Decision Confidence: Explanations help users trust the system and understand why certain locations are recommended.
  • Better Personalisation: Feedback (like/dislike) and search history improve future recommendations over time.
  • Context-Aware Suggestions: Recommendations consider where the user is and how they explore the map.

Target Users & Use Cases

  • Everyday Explorers: Students or young adults looking for cafes, study places, and hangout spots.
  • Visitors & Tourists: People exploring new areas who need location-aware recommendations with clear explanations.
  • Business Partners: Businesses that want their POIs to appear in a smarter, more transparent recommendation platform.
  • System Administrators: Operators who need analytics, moderation tools, and system health dashboards.

Future Potential & Extensions

Beyond this FYP, SpatiaLynk can be extended towards:

  • Smart city planning and urban mobility services.
  • Integration with tourism boards and local business networks.
  • More advanced natural-language explanations using NLG models.
  • Cross-platform deployment with deeper analytics for business partners and administrators.
